## рҹҺҜ The Problem & Solution

### The Problem
Traditional payment rails (Stripe, PayPal) have a **~30Вў minimum** per transaction. This makes micro-tasks economically impossible to payment-enable:

| Task | Cost | Feasible? |
|------|------|------------|
| 1 AI inference | $0.009 | вқҢ (Stripe: $0.30 min) |
| 1 second of compute | $0.001 | вқҢ |
| Micro-data transfer | $0.0001 | вқҢ |

### The Solution
**Circle Nanopayments + Arc Testnet + x402** enables true **fractional micropayments**:

```mermaid

    subgraph Traditional
        T1[$0.30 min] -----> |Impossible| T2[Micro-tasks]
    end
    
    subgraph Pulse
        P1[$0.009] -----> |вң“ Works| P2[AI Inference]

        P1 -----> |вң“ Works| P4[Data Transfer]
    end

    style T1 fill:#666,color:#fff
```

## рҹҡҖ Quick Start

### 1. Install
```bash
git clone https://github.com/shikhar-pulse/pulse.git
cd pulse
npm install
cd frontend && npm install && cd ..
```

### 2. Configure
```bash
# .env file
CIRCLE_API_KEY=your_key
CIRCLE_ENTITY_SECRET=your_secret
NEXT_PUBLIC_API_URL=http://localhost:3001
```

### 3. Run
```bash
npm run dev
```

- **Frontend:** http://localhost:3000
- **API:** http://localhost:3001

### 4. Demo Mode
```bash
# Generates 200+ on-chain transactions
npm run demo
```

## рҹ”җ Security

- **EIP-712 Signing** вҖ” Cryptographic proof of work
- **JWT Authentication** вҖ” Secure session management
- **Circle MPC Wallets** вҖ” No private key exposure
- **Budget Guards** вҖ” Prevent overspending
- **Idempotency Keys** вҖ” Prevent duplicate payments
